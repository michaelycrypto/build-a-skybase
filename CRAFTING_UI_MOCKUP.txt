USER INTERACTION FLOW:

┌────────────────────────────────────────────────────────────────────────────────┐
│                                                                                │
│  1. Press [E] → Inventory Opens                                               │
│                                                                                │
│  2. Crafting Panel Auto-Updates                                               │
│     ┌─────────────────────────────────────────┐                               │
│     │ Check Inventory for Each Recipe        │                               │
│     │   ├─ Has 1+ Oak Log? → Show Oak Planks │                               │
│     │   ├─ Has 2+ Planks? → Show Sticks      │                               │
│     │   └─ Has Materials? → Show Tool Recipe │                               │
│     └─────────────────────────────────────────┘                               │
│                                                                                │
│  3. Player Hovers Recipe Card                                                 │
│     → Background lightens (if craftable)                                      │
│                                                                                │
│  4. Player Clicks [►] Craft Button                                            │
│     ┌─────────────────────────────────────────┐                               │
│     │ Validate Materials                      │                               │
│     │ ├─ Consume Inputs                       │                               │
│     │ ├─ Add Outputs                          │                               │
│     │ ├─ Play Sound Effect                    │                               │
│     │ └─ Refresh UI                           │                               │
│     └─────────────────────────────────────────┘                               │
│                                                                                │
│  5. Inventory Updates → Crafting Panel Refreshes                              │
│     → Recipe availability recalculated                                        │
│                                                                                │
│  6. Press [E] or [X] → Inventory Closes                                       │
│                                                                                │
└────────────────────────────────────────────────────────────────────────────────┘

│                                                                                │
│  When Inventory Changes:                                                      │
│  ┌────────────────────────────────────────────────────────────┐               │
│  │ 1. Player picks up/drops items                             │               │
│  │ 2. Player crafts something                                 │               │
│  │ 3. Player moves items between slots                        │               │
│  └────────────────────────────────────────────────────────────┘               │
│                           ↓                                                    │
│  ┌────────────────────────────────────────────────────────────┐               │
│  │ ClientInventoryManager:NotifyInventoryChanged()            │               │

│  ┌────────────────────────────────────────────────────────────┐               │
│  │ CraftingPanel:RefreshRecipes()                             │               │
│  │   ├─ Count all items in inventory                          │               │
│  │   ├─ Check each recipe for availability                    │               │
│  │   ├─ Update card states (enabled/disabled)                 │               │
│  │   └─ Update max craft counts                               │               │
